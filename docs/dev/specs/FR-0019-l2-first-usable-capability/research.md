# FR-0019 研究记录

## 研究问题

在未知网站没有现成适配器时，如何定义“先做成一次”的正式边界，使这次成功既不是一次性实验，也不会被误报成已经正式可复用。

## 当前输入

- `docs/dev/roadmap.md` 中 Phase 2 的 L2 首次可用能力
- `docs/dev/architecture/system-design/reference.md` 中 Phase 2 L2 首次可用占位章节
- `#157` issue 范围
- `FR-0017` 对候选能力 handoff 的需求

## 收敛结论

1. L2 首次可用当前最缺的是正式的成功判定与 handoff 输出，而不是更多临时探索命令。
2. “先做成一次”必须与“已验证”“已正式复用”严格分层。
3. L2 首次可用输出必须能进入 `FR-0017`，否则 Phase 2 主闭环无法成立。
4. L2 失败时需要稳定的 L1 fallback 建议条件，但不应在本 FR 中把 L1 细节做完。
