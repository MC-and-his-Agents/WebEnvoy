# 中等事项设计说明模板

> 适用范围：影响多个文件或模块，但不改变上位架构边界、共享契约、共享数据模型，也不属于高风险链路的事项。
>
> 默认优先把这些字段写进 PR 描述；只有在需要跨会话冻结输入、承载 review 讨论或给后续实现复用时，才单独落成 Markdown。

## 基本信息

- 关联 Issue：
- 关联 PR：
- 负责人：

## 背景

一句话说明为什么现在要做这件事。

## 目标

- 这次明确交付什么

## 非目标

- 这次明确不交付什么

## 范围

- 受影响模块 / 文件
- 受影响命令 / 页面 / 流程

## 方案摘要

- 核心改动点
- 为什么选择这条路径

## 影响面与风险

- 可能影响哪些已有行为
- 为什么仍属于中等事项，而不是正式 FR / 高风险事项

## 验证方式

- 本地验证命令
- 需要观察的关键结果

## 回滚方式

- 如需回退，如何快速收口

## 升级信号

出现以下情况时，应停止按中等事项推进并升级为正式 FR：

- 触及上位架构边界
- 触及共享契约或共享数据模型
- 出现高风险执行链路、副作用或回滚复杂度明显上升
- 范围扩张到无法通过单一 PR 清晰收口
